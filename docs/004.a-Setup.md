# Fixify — Environment & Dependency Setup

## 1. Create your `.env` file

Copy `.env.example` to `.env` at the project root and fill in your real values:

```
SUPABASE_URL=https://YOUR_PROJECT_ID.supabase.co
SUPABASE_ANON_KEY=eyJhbGci...your-real-key
GOOGLE_MAPS_API_KEY=AIzaSy...your-real-key
```

**IMPORTANT — add `.env` to `.gitignore` immediately:**

```
# .gitignore
.env
```

---

## 2. Update `pubspec.yaml`

Add these dependencies and the `.env` asset:

```yaml
dependencies:
  flutter:
    sdk: flutter

  # Existing
  supabase_flutter: ^2.3.4
  flutter_animate: ^4.5.0
  google_fonts: ^6.2.1

  # Add these new ones:
  flutter_dotenv: ^5.1.0 # reads .env file
  image_picker: ^1.0.7 # photo upload in request service
  google_maps_flutter: ^2.5.0 # Google Maps on location step
  geolocator: ^11.0.0 # get device GPS position
  geocoding: ^3.0.0 # reverse geocode latlng → address

flutter:
  uses-material-design: true
  assets:
    - .env # ← ADD THIS
    - assets/images/
    - assets/icons/
```

Then run:

```bash
flutter pub get
```

## 3. Google Maps native setup

### Android — `android/app/src/main/AndroidManifest.xml`

Inside `<application>`, add:

```xml
<meta-data
    android:name="com.google.android.geo.API_KEY"
    android:value="${GOOGLE_MAPS_API_KEY}" />
```

Then in `android/app/build.gradle`, inside `defaultConfig`:

```gradle
manifestPlaceholders = [
    GOOGLE_MAPS_API_KEY: System.getenv("GOOGLE_MAPS_API_KEY") ?: ""
]
```

**Simpler approach for MVP** — just paste the key directly in AndroidManifest.xml  
(never commit to public repos):

```xml
<meta-data
    android:name="com.google.android.geo.API_KEY"
    android:value="AIzaSy...your-real-key" />
```

### iOS — `ios/Runner/AppDelegate.swift`

```swift
import GoogleMaps

@UIApplicationMain
@objc class AppDelegate: FlutterAppDelegate {
  override func application(
    _ application: UIApplication,
    didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
  ) -> Bool {
    GMSServices.provideAPIKey("AIzaSy...your-real-key")
    GeneratedPluginRegistrant.register(with: self)
    return super.application(application, didFinishLaunchingWithOptions: launchOptions)
  }
}
```

## 4. image_picker Android permissions

In `android/app/src/main/AndroidManifest.xml`, inside `<manifest>`:

```xml
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
```

For Android 13+, also add:

```xml
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES"/>
```

## 5. booking_photos Supabase bucket

Run this in your Supabase SQL editor to create the photo storage bucket:

```sql
INSERT INTO storage.buckets (id, name, public)
VALUES ('booking_photos', 'booking_photos', false);

CREATE POLICY "Customers can upload booking photos"
  ON storage.objects FOR INSERT
  WITH CHECK (
    bucket_id = 'booking_photos'
    AND auth.uid()::text = (storage.foldername(name))[1]
  );

CREATE POLICY "Users can view booking photos"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'booking_photos' AND auth.role() = 'authenticated');
```
